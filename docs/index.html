GA4 Analytics Engineering with R, Arrow & DuckDB & Parquet

A practical guide to downloading, storing, and analyzing Google Analytics 4 event data

Overview

Google Analytics 4 (GA4) stores event data in deeply nested, hierarchical structures in BigQuery — a format optimized for collection, but challenging for analysis. This series of notebooks walks through the full analytics engineering workflow: from understanding the raw nested structure, to downloading and storing data efficiently, to answering real marketing questions at scale.

CautionCaution for Coding Accuracy

You are advised to be aware of potential inaccuracy of the coding provided on this site. Depending on the computer setup, IDE you used, packages you installed, how you want to save you your GA4 data, some adjusmtents are expected.

Also, due to limitations of publicl GA4 data, some data is blank (e.g., user properties), leading to no insights for the variables in the demonstration.

Warning

Working with large data makes you slow. Saving data in parquet solve the problem. However, GA4’s deeply nested data present challenges in saving them in your local computer. All the demonstrations were made on a relatively small GA4 data (i.e., 7 days) with about 400,000 rows of events. You are expected to save the file in eihter RDS or qs files on your computer.

Even though they are not big, it took very long time to flatten event params data when RDS or qs data were read into RAM.

To solve this problem, it is worth looking into Data Acquisition Guide for GA4 Public Data

What You Will Learn

  • Rectangling — how to flatten nested list-columns using tidyr verbs like unnest_longer(), unnest_wider(), and hoist()
  • GA4-specific challenges — handling union-type value structs, key-value event_params arrays, and BigQuery INT64 types
  • Efficient storage — comparing RDS, qs2, Parquet, Arrow, and DuckDB for local caching of GA4 data
  • Large-scale analytics — using Arrow and DuckDB to query GA4 data without loading everything into RAM
  • Marketing analytics — translating business questions into R code using GA4 event data

The GA4 Data Challenge

GA4 event data from BigQuery is not a flat table. Every row contains nested structs and repeated arrays:

Column Structure Challenge
event_params Key-value struct array Requires 3-stage unnesting pipeline
items Repeated struct array Requires combined unnest_longer() + unnest_wider()
device Nested struct Requires unnest_wider() or hoist()
geo Nested struct Requires unnest_wider() or hoist()
traffic_source Nested struct Requires unnest_wider() or hoist()
ecommerce Nested struct Requires unnest_wider() or hoist()
